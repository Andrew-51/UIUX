What is UI?

User Interface (UI) is everything a user interacts with when using a digital product.

Visual Elements

Response Time

  • The visual layer that users see and interact with
  • Encompasses buttons, forms, typography, colors, layouts, and animations

  • Follows design systems and visual hierarchies
  • Transforms product requirements into accessible interfaces

What is UX?

User Experience (UX) is how users feel when interacting with a product.

  • The entire journey and emotional response of users
  • Encompasses research, testing, structure, and usability
  • Focused on functionality and problem-solving
  • Aims to create meaningful, relevant, and valuable experiences
  • Based on deep understanding of user behaviors and motivations

  • Establishes brand identity and recognition
  • Communicates functionality through visual cues

Responsive Design Best Practices

DO

  • Design fluid layouts that adapt to screen sizes
  • Use relative units (em, rem, %) instead of pixels

  • Prioritize content differently across devices
  • Test on actual devices, not just simulators

DON'T

  • Hide critical information on mobile
  • Rely on hover states for mobile interfaces

Intuitive Gesture Design

Touch interactions should feel natural and discoverable

DO

  • Use standard gestures consistently
  • Provide visual cues for available gestures
  • Ensure touch targets are at least 44×44 points
  • Test gestures with real users
  • Consider edge cases and accessibility

DON'T

  • Invent new gestures without clear need
  • Hide essential functions behind gestures
  • Create gesture conflicts
  • Make touch targets too small or too close
  • Rely solely on gestures without alternatives

Purposeful Animations

Animations should enhance the experience, not distract from it

DO

  • Use animations to convey meaning
  • Keep transitions smooth and quick (150-300ms)
  • Animate along logical paths
  • Ensure animations are consistent
  • Provide option to reduce motion

DON'T

  • Add animations purely for decoration
  • Create animations that are too slow
  • Use jarring or unexpected movements
  • Animate everything on the screen at once
  • Ignore accessibility concerns

Accessibility Matters

DO

  • Use sufficient color contrast
  • Support screen readers
  • Provide text alternatives for images
  • Design for keyboard navigation
  • Make touch targets adequately sized

DON'T

  • Rely solely on color to convey information
  • Use tiny text or fonts below 16px
  • Create interfaces that require precision movements
  • Autohide critical navigation
  • Ignore accessibility guidelines

Accessibility Foundations

Creating experiences that work for everyone is not optional — it's essential

1.3 billion people globally live with some form of disability

  • Accessibility benefits everyone, not just people with disabilities
  • Improves usability across different contexts and situations
  • Legally required in many countries and industries
  • Expands your potential user base significantly
  • Creates a more inclusive digital world

Key Principles

  • Perceivable: Information must be presentable in ways all users can perceive
  • Operable: UI components must be operable by everyone
  • Understandable: Information and operation must be understandable
  • Robust: Content must be robust enough to work with assistive technologies

Color Contrast & Accessibility

Sufficient contrast ensures content is perceivable by everyone

DO
Primary Button (4.5:1 contrast ratio)
High Contrast Text (21:1 contrast ratio)
Light Text on Dark (15:1 contrast ratio)

WCAG Guidelines:

  • Text: Minimum 4.5:1 contrast ratio
  • Large Text: Minimum 3:1 contrast ratio
  • UI Components: Minimum 3:1 contrast ratio
DON'T
Low Contrast Button (1.2:1 contrast ratio)
Gray on Light Gray (1.5:1 contrast ratio)
Faded Text on Dark (2.1:1 contrast ratio)

Insufficient contrast makes content:

  • Difficult to read for users with low vision
  • Unusable in bright lighting conditions
  • Problematic for users with color vision deficiency

Keyboard Accessibility

Many users navigate exclusively with keyboards

DO

  • Ensure all interactive elements are keyboard accessible
  • Provide visible focus indicators
  • Maintain logical tab order matching visual layout
  • Create keyboard shortcuts for common actions
  • Test your interface using only a keyboard

DON'T

  • Create interfaces that require mouse hovering
  • Remove focus outlines without alternatives
  • Create keyboard traps users can't escape
  • Hide important controls behind complex gestures
  • Rely on drag-and-drop without alternatives

Screen Reader Support

Make your content accessible to people who can't see it

Key Techniques

  • Provide meaningful alt text for images
  • Use semantic HTML elements
  • Add ARIA labels for custom elements
  • Ensure form fields have proper labels
  • Create descriptive link text

HTML Example

<!-- Bad -->
<div onclick="submit()">Click here</div>

<!-- Good -->
<button type="submit" aria-label="Submit form">

Touch Target Size

Ensure interactive elements are large enough for everyone to use

Recommended Sizes

  • Minimum: 44×44 pixels (WCAG)
  • Apple: 44×44 points
  • Google: 48×48 dp minimum
  • Microsoft: 40×40 pixels
  • Add sufficient spacing between targets (8px minimum)

Touch targets need to accommodate users with:

  • Motor control limitations
  • Larger fingers
  • Tremors or movement disorders
DO

Adequate target size with proper spacing between controls

DON'T

Targets too small and spaced too closely together

Text Alternatives

Provide alternatives for all non-text content

DO

  • Write descriptive alt text for images
  • Include captions for videos
  • Provide transcripts for audio content
  • Use ARIA labels for interactive icons
  • Create text alternatives for complex visualizations

DON'T

  • Use generic alt text (e.g., "image")
  • Include text as part of images
  • Omit descriptions for decorative images
  • Create infographics without text alternatives
  • Use color alone to convey information

Semantic Content Structure

Properly structured content enhances comprehension for all users and is essential for assistive technology

DO
<header> <h1>Annual Report 2025</h1> </header> <nav> <ul> <li><a href="#overview">Overview</a></li> <li><a href="#performance">Performance</a></li> </ul> </nav> <main> <section id="overview"> <h2>Company Overview</h2> <p>Financial summary for 2025...</p> </section> </main>

Screen reader announces: "Annual Report 2025, heading level 1"

Semantic HTML provides structure and meaning

DON'T
<div class="header"> <div class="title">Annual Report 2025</div> </div> <div class="menu"> <div class="item">Overview</div> <div class="item">Performance</div> </div> <div class="content"> <div class="section"> <div class="section-title">Company Overview</div> <div>Financial summary for 2025...</div> </div> </div>

Screen reader announces: "Annual Report 2025"

Generic divs lack semantic meaning for assistive technology

Heading Hierarchy

H1: Page Title
H2: Section
H3: Subsection
H4: Component
• Screen readers build navigation outlines
• Users navigate via heading structure
• Never skip heading levels

Benefits of Proper Structure

  • Assistive Technology Navigation

    Screen reader users can jump between sections and understand content organization

  • Enhanced SEO

    Search engines better understand your content's organization and importance

  • Code Maintainability

    Developers can more easily understand and update well-structured code

  • Content Adaptability

    Properly structured content can adapt to different devices and contexts
