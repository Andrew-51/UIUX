Core Principles of Good Design

  • Clarity: Design should be intuitive with clear visual hierarchy
  • Consistency: Similar elements should behave similarly
  • Feedback: Users should know when actions are successful
  • Efficiency: Accomplish tasks with minimum steps
  • Forgiveness: Allow users to undo mistakes easily
  • Accessibility: Design for all users, regardless of ability

UX: DOs and DON'Ts

DO

  • Conduct user research before designing
  • Create clear user flows & journeys
  • Test designs with real users
  • Design for accessibility from the start
  • Use data to inform design decisions

DON'T

  • Design based on assumptions alone
  • Sacrifice usability for aesthetics
  • Copy competitors without understanding why
  • Ignore user feedback and test results
  • Create complex flows for simple tasks

UI: DOs and DON'Ts

DO

  • Maintain visual hierarchy & clear focal points
  • Use whitespace effectively
  • Ensure sufficient color contrast
  • Use consistent UI patterns
  • Make interactive elements obvious

DON'T

  • Overwhelm with too many colors or fonts
  • Use tiny touch targets on mobile
  • Hide important features in hamburger menus
  • Create cluttered, busy interfaces
  • Use low contrast text that's hard to read

Designing for Accessibility

Perceivable

  • Alt text for images
  • Color contrast (4.5:1 minimum)
  • Text resizing without breaking layout

Operable

  • Keyboard navigation
  • Sufficient time to read content
  • No flashing content

Understandable

  • Clear labels and instructions
  • Consistent navigation
  • Error prevention & suggestions

Robust

  • Compatible with assistive technologies
  • Valid, semantic HTML
  • ARIA when necessary
